How It Works

RLL (Ralph-Lisa Loop) is a dual-agent collaboration system where two AI agents work together in a turn-based review cycle — just like a developer and code reviewer.

Architect Analyzes failing tests,
writes requirements
→
Ralph (Developer) Claude Sonnet 4.6
Plans, codes, tests
→
Lisa (Reviewer) Codex GPT-5.4
Reviews, verifies, feedback
→
Consensus Both agree the fix
is correct

Each task's detail page shows the complete Ralph-Lisa dialogue — you can see exactly how the AI agents discuss, debug, and refine their solution. Click any task above to view.

Why These Metrics?

Solve Rate — Percentage of completed tasks where all target tests pass (gap→0). In CI/CD terms: the pipeline goes green. Production doesn't give partial credit — code either works or it doesn't.

Zero-Regression Rate — Percentage of tasks where test pass counts never decreased across epochs. Measures whether the AI agent is safe to trust with your codebase. The SWE-CI paper reports that most single-agent models achieve below 25%, with only Claude Opus exceeding 50%. RLL internalizes code review into the agent loop — Lisa (reviewer) catches regressions before they persist, similar to how CI pipelines protect human teams.
